# FRONTEND GUIDELINES

## 作用

这份文档只回答：前端怎么把结果展示成作品，而不是后台页面。

## 前端目标

- 让人快速看懂系统是做什么的
- 让人直观看懂识别结果
- 稳定承接后端统一结果协议

## 页面顺序

1. 说明系统能力
2. 发起分析任务
3. 展示原图和 `Mask`
4. 展示病害详情
5. 做筛选、对比和导出
6. 回看历史结果

## 信息层级

- 第一层：系统能力
- 第二层：当前任务状态
- 第三层：核心识别结果
- 第四层：扩展信息

## 页面规则

- 上传页要少参数、入口明显、先讲输出
- 首页只做说明和入口，不堆复杂功能
- 结果页必须是主画面 + 侧边信息面板
- `Mask`、病害列表、置信度、耗时必须一屏可见
- 结果详情要支持联动高亮
- 批量页要突出进度和结果下载
- 历史页要支持搜索、筛选、排序、分页、对比

## UI 规则

- 信息清晰优先于炫技
- 桌面优先，同时兼顾移动端查看
- 图像区域优先级高于装饰元素
- 参数少而明确
- 状态变化要有即时反馈
- 错误信息要能被普通用户理解

## 视觉规则

- 使用深色蓝青系产品化界面
- 用蓝青色作为主强调色
- 红/橙只用于病害高亮或风险提示
- 保留玻璃感、微光和层次感
- 结果页要优先呈现“主图 + 结论 + 量化信息”

## 状态

- 未上传
- 上传中
- 推理中
- 推理成功
- 推理失败
- 批量处理中
- 批量部分失败

## 协议

前端只消费统一结果协议，不解析模型原始输出。

默认字段：

- `image_id`
- `inference_ms`
- `model_version`
- `detections`
- `category`
- `confidence`
- `bbox`
- `mask`
- `metrics`

## LLM诊断报告渲染

前端使用 `react-markdown` 渲染后端返回的LLM智能诊断报告，支持：

- 流式markdown输出（实时显示生成内容）
- 标准markdown格式（标题、列表、表格、代码块）
- 业务专家口吻的定损分析文本

## 结论

前端的任务不是“把结果画出来”，而是把识别能力转成用户能理解、评审能看懂、系统能稳定演示的产品体验。
