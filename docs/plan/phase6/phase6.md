# Phase 6

## 阶段主题

业务稳定性与决策能力优化

## 阶段目标

将系统从“可运行的业务流程原型”升级为“企业可落地的巡检运营系统”。

本阶段重点从“页面功能齐全”转向“真实团队可长期高效使用”：

- 稳定性：任务链路可靠、失败可恢复、状态可追踪
- 决策性：识别结果可映射标准口径并转成可执行优先级
- 运营性：前端以处置效率为核心，不以展示为核心
- 集成性：可与企业已有工单与资产系统联动

## 当前状态

`Phase 6` 以优化与收敛为主线定义。

当前已经具备：

- `batch workflow` 主链路
- 异步任务执行与失败重试
- 检测检索、人工复核、预警事件基础闭环
- 企业化 dashboard 基础入口与核心页面

当前尚未具备：

- 成体系的稳定性指标与性能基线
- 标准化评级与优先级规则引擎
- critical finding 时效处置模板
- 对外工单系统对接的最小接口规范

## 现有系统深度诊断（不脱离当前代码）

1. 信息架构现状
   - 已形成 `ops` 主导航与多页面工作台。
   - 历史能力存在，但在企业工作流中的入口层级需明确，避免“有功能但难发现”。

2. 后端链路现状
   - 数据已从单次结果文件存储升级为关系型链路存储。
   - 但“任务质量”指标（成功率、恢复率、时延、积压）尚未成为一等对象。

3. 决策支持现状
   - 具备检测与预警，但“标准化评级 + 时限建议 + 升级策略”尚未固化为规则引擎。

4. 企业落地差距
   - 缺统一 SLA 概念和处置看板。
   - 缺跨系统对接最小协议（工单、导出、状态回写）。
   - 缺面向管理层复盘的指标沉淀（周/月质量趋势）。

## 主要任务

1. 稳定性优化
   - 建立任务幂等与重复提交保护策略。
   - 优化失败分类与重试策略（可重试/不可重试分层）。
   - 补齐任务执行超时、队列积压和慢任务监控字段。
   - 新增质量指标聚合接口（成功率、恢复率、P50/P95、积压量）。

2. 决策规则优化
   - 建立标准化评级字段与规则引擎版本管理。
   - 输出风险等级、处置优先级和建议时限。
   - 先实现中国 JTG 口径，保留多口径扩展位。
   - 引入规则解释字段（命中规则、阈值、版本号），确保可解释可审计。

3. 关键告警机制优化
   - 增加 critical finding 规则模板。
   - 建立告警处置 SLA 字段（触发时间、响应时间、关闭时间）。
   - 支持按“超时/临近超时”筛选和优先处理。
   - 增加升级策略（超时自动升级、重复出现加权升级）。

4. 前端运营效率优化
   - 总览页增加高风险待办、复核积压、告警时效卡片。
   - 批次与告警页强化批量操作与快捷筛选。
   - 构建“3分钟定位高风险并处置”的交互路径。
   - 明确“历史记录”在 `ops` 体系中的一级或二级入口策略，保证追溯路径稳定。

5. 集成与导出优化
   - 定义最小工单接口（创建、状态回写、附件引用）。
   - 统一结构化导出字段，满足归档和对接需求。
   - 形成对接说明文档与字段字典。

## 企业视角下的目标用户与操作路径

1. 运营值班人员（日常高频）
   - 关注：今天是否有超时风险、哪些批次阻塞、哪些告警必须先处理。
   - 页面主路径：`ops/overview -> ops/alerts -> ops/reviews`。

2. 复核工程师（结果纠偏）
   - 关注：高置信但高风险项、模型误报漏报、复核积压。
   - 页面主路径：`ops/search -> ops/items/[id] -> ops/reviews`。

3. 管理负责人（周/月复盘）
   - 关注：任务稳定性、风险趋势、SLA 达成率、闭环效率。
   - 页面主路径：`ops/overview + 导出报表 + 历史档案`。

## 阶段分解（建议按三轮推进）

1. 第一轮：稳定性与数据可信
   - 输出稳定性指标接口与基础看板。
   - 完成任务幂等、失败分类、超时治理。
   - 验收：高并发下任务链路可持续运行且可定位问题。

2. 第二轮：决策与处置闭环
   - 上线规则引擎 v1（JTG）与 critical finding。
   - 前端形成高风险优先与批量处置流。
   - 验收：从识别结果到处置动作形成标准闭环。

3. 第三轮：企业对接与管理复盘
   - 工单接口、导出规范、字段字典、错误码文档落地。
   - 补齐历史追溯入口与复盘视图一致性。
   - 验收：可与外部系统联调并支持管理侧复盘。

## 推荐执行顺序

1. 先建立指标基线，再做稳定性修复。
2. 先固化规则模型，再做前端决策展示。
3. 先跑内部闭环，再推进外部系统集成。
4. 每次优化必须绑定可量化验收指标。

## 阶段产出

- 一套稳定性优化方案与运行指标面板。
- 一套版本化的评级与优先级规则引擎。
- 一套 critical finding 事件模板与时效追踪机制。
- 一套面向运营处置效率的 dashboard 优化页面。
- 一套最小可用工单对接接口与导出规范。
- 一套历史追溯入口与 `ops` 工作流统一方案。

## 验收标准

- 任务成功率、重试恢复率、端到端时延具备可视化与基线。
- 系统可稳定输出风险等级与处置优先级建议。
- critical finding 事件可被触发、跟踪、关闭并记录时效。
- 前端可快速定位高风险项并完成批量处置。
- 导出与工单接口可用于外部系统联调。
- 历史档案入口稳定可见，支持按桥梁/批次/时间回溯。
