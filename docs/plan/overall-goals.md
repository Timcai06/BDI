# 整体目标

## 项目总目标

本项目的总目标是构建一个面向无人机桥梁巡检场景的病害识别系统原型。

该系统不是单一的模型 demo，而是一个以 `YOLOv8-seg` 为当前核心、能够兼容后续优化算法、并具备完整展示与导出能力的产品化系统。

## 核心方向

### 1. 以 YOLOv8 为当前核心

第一阶段以 `YOLOv8-seg` 作为主要推理引擎，优先保证可运行、可联调、可展示。

### 2. 兼容后续优化算法

系统必须具备兼容基于 `YOLOv8` 优化后模型的能力，确保算法同学后续升级模型时，前后端不需要频繁返工。

### 3. 建立完整产品闭环

系统需要完成从图像输入、模型推理、结果展示到结果导出的完整流程，而不是停留在训练脚本和实验结果层面。

### 4. 服务比赛目标

项目最终交付应同时体现：

- 核心算法系统
- 实际应用效果展示
- 结构化结果输出
- 工程完整性

## 初步验收成果

第一阶段完成后，应能达到以下验收结果：

- 能上传单张桥梁图像
- 能接入当前 `YOLOv8-seg`
- 能输出病害类别、Mask、置信度
- 能展示原图与叠加图
- 能导出 JSON 和可视化图
- 系统结构上已支持后续优化模型接入

## 开发原则

- 先完成 MVP，再逐步增强
- 先建立统一协议，再扩展模型版本
- 先本地单机原型，再考虑更复杂部署
- 先保证前后端与算法协作稳定，再追求高级优化
