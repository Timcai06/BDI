# Phase 2 执行说明

## 目标

本阶段的目标是把现有文档规划落成一个最小可运行的系统骨架，优先完成单张图像识别的前后端闭环。

## 范围

本阶段仅承诺以下能力：

- `FastAPI` 后端骨架
- `Next.js` 前端骨架
- 统一请求与结果协议
- `GET /health`
- `POST /predict`
- 单张图像上传与结果展示
- 标准 JSON 结果导出

以下内容只做结构预留，不纳入本阶段完成定义：

- 批量任务
- 历史任务页
- 多模型切换 UI
- `sliced` 推理真实执行
- 云端存储或 BaaS

## 实施顺序

1. 后端协议与 mock 接口先落地
2. 前端页面与状态流基于 mock 接口联调
3. 模型适配层接入真实 `YOLOv8-seg`
4. 本地文件产物与导出能力补齐
5. 进行构建、测试和端到端联调验收

## 关键约束

- 前端只消费统一结果协议，不读取模型原始输出
- 后端通过适配层调用模型，不把 Ultralytics 原始结果暴露给 API
- 本阶段默认本地文件存储，不接数据库
- 协议字段应优先保持稳定，为后续模型升级留出空间

## 完成定义

满足以下条件视为 `Phase 2` 完成：

- 本地可启动前后端开发服务
- 上传单张图片可得到结构化结果
- 结果页可展示病害列表、置信度、推理耗时和基础说明
- 后端能返回统一错误结构
- 标准 JSON 结果可保存到本地
- 根目录与 `AGENTS/04-memory` 可回写当前进展

## 当前结论

`Phase 2` 已完成。

已确认：

- 真实 `YOLOv8-seg` runner 已完成单图联调验收
- 浏览器侧上传到结果展示的完整链路已跑通
- 前端可直接消费真实返回结果，无需协议调整
- 标准 JSON 与 overlay 产物已完成联调验证
- 项目已具备进入 `Phase 3` 的前置条件
