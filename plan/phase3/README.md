# Phase 3 执行说明

## 目标

本阶段的目标是在 `Phase 2` 已完成真实单图闭环的基础上，把系统从“能跑通”提升到“能展示、能解释、能回看、能稳定演示”。

## 开工前提

进入 `Phase 3` 前，默认以下事项已经完成并验收：

- 浏览器侧单图上传到结果展示的完整链路已跑通
- 后端已可调用真实 `YOLOv8-seg` runner
- 前端已可直接消费真实返回结果，无需协议改动
- 标准 JSON 与 overlay 产物已生成并完成联调验证
- 基础错误路径已完成验证

## 范围

本阶段优先承诺以下能力：

- 更完整的结果展示页
- 病害详情展示增强
- 筛选与结果联动交互
- overlay / JSON 结果导出入口
- 统一错误状态展示
- 本地历史结果与回看基础能力

以下内容仍不纳入本阶段完成定义：

- 批量任务正式流程
- 多模型切换 UI
- 高分辨率 `sliced` 推理真实执行
- 数据库或云端结果管理

## 实施顺序

1. 先收敛结果页展示信息层级与导出入口
2. 再补齐错误态、空态和异常反馈
3. 再增加历史结果读取与回看入口
4. 最后做演示稳定性校验与文档回写

## 关键约束

- 前端继续只消费统一结果协议，不依赖模型私有字段
- 历史结果优先复用本地文件产物，不提前引入数据库
- 展示增强优先围绕“演示清晰度”和“可解释性”，避免过早扩展复杂业务流
- 本阶段完成后，系统应具备稳定单图演示能力，为 `Phase 4` 的批量与多模型能力打基础

## 完成定义

满足以下条件视为 `Phase 3` 完成：

- 结果页可清楚展示检测效果、病害详情和关键指标
- 用户可导出至少 JSON 与 overlay 两类结果
- 错误态、空态和失败原因可被清楚展示
- 本地历史结果可被基础回看
- 根目录与 `AGENTS/04-memory` 已同步更新为最新阶段状态
