# Phase 5

## 阶段主题

工程闭环建设与大模型(LLM)智能定损

## 阶段目标

将系统从“纯视觉算法原型”升级为**“具备端到端量化分析与大模型(LLM)智能定损能力的桥梁巡检微 SaaS 平台”**。

本阶段的核心策略是**降维打击**：不纠结于纯 CV 算法（YOLO/Mask R-CNN）的微小指标提升，而是通过极其完善的应用体验、物理量化数据以及自动化专家报告生成，体现项目的工程落地价值和商业产品化能力。

## 当前状态

`Phase 5` 刚刚启动，前置的多模型注册体系（Phase 4）已基本建立，下一步重心从后端算法调度扩展到前台产品闭环与大模型整合。

## 主要任务

1. 引入 LLM 分析节点构建智能诊断引擎
   - 申请外部大模型 API Key（如通义千问、DeepSeek）。
   - 在 FastAPI 新增 `/api/generate-diagnosis` 路由，拼接 YOLO 吐出的结构化数据（如裂缝长度、面积）生成 Prompt，请求大型语言模型，获得专家的诊断摘要与养护建议。
   - 前端 Dashboard 增加“AI 专家诊断”区域，运用流式（Streaming）请求实现炫酷的科技感打字机特效。

2. 像素级量化转换与物理映射（赛题核心）
   - 在前端预留 `GSD (Ground Sample Distance)` 或焦距/飞行高度换算系数的调整滑块。
   - 后端使用 OpenCV 获取 Mask 多边形的物理尺寸计算结果，实现前后端的参数流转。
   - 实现区域“面积占比”的计算闭环，为模型判别“轻度/重度伤害”提供量化的支撑。

3. 干扰项过滤证明层体系（针对性赛题演示）
   - 利用视觉模型主动返回判定为“阴影、接缝、污点”的低置信度疑似物（附带 `interference: true` 分类）。
   - 前端将其以低调的半透明灰色渲染，作为系统具备优秀“抗干扰能力”的可视化自证。

4. 性能指标透明拆分
   - 打破原先单一整合的 `inference_ms`，将其分解为预处理 (`preprocess`)、模型推理 (`model`) 和后处理 (`postprocess`)。
   - 在前端性能表盘中明示“单张 4K 图片纯模型推理 < 200ms”，直接向评委展示技术达标点。

5. 企业级检验报告一键导出引擎（闭环终点）
   - 前端集成 PDF 工具库（如 jsPDF + html2canvas 或定制的打印样式），将原图、检测结果图表、病害清单和 LLM 总结合并到一份标准制式的报告单据中。
   - 实现真正的“看图即得病历”，满足评委对外包服务应用的最高期望。

## 推荐执行顺序

1. **先跑通 LLM 分析节点链路**：因为这是最容易出视觉效果、也是降维打击感最强的部分，最容易提振开发士气。
2. **再搞定物理结构量化 (GSD系数与算法重构)**：这是赛题硬性要求，需要扎实的数据流传递。
3. **增加 PDF 报告一键生成与排版优化**：确保数据出来后能够落地为可分享的文档资料。
4. **最后增添锦上添花的细节**：包括性能耗时拆解展示和抗干扰可视化图层隔离。

这样做可以保证我们在面对验收时，永远呈现出主次分明、最大化亮眼特性（LLM生成+精美PDF）的产品姿态，即便没有时间完成边缘项也不影响主线展示。

## 阶段产出

- 整合外部大模型 API 的分析路由。
- 具有流体美学极客风格、带流式打字输出面板的重构 Dashboard 前端。
- 动态的量化尺寸转化器和交互表盘。
- 支持一键生成图文并茂的体检 PDF 报告下载体系。
- 更加深度的参数解析视图（含性能细分耗时与被拦截干扰项标识）。

## 验收标准

- 在前端上传病害图后，能在侧边栏流式打出有专家口吻的分析文字。
- 界面能够调节像素尺寸转换系数，并实时观察到底部病害长宽数据跟随着变化（从 px 变化为 mm）。
- 后台能正确分辨真正的病害推演时间与图片裁剪时间。
- 点击“生成报告”按钮，浏览器下载了一份完整的、带有分析和图例 A4 排版格式 PDF 源文件。
