# Phase 5 执行说明

## 目标

本阶段的目标是将系统从“视觉算法 Demo”升级为“具备端到端量化分析与大模型(LLM)智能定损能力的桥梁巡检微 SaaS 平台”。通过引入极客 UI、真实的物理尺寸映射体系以及一键式的专家 PDF 报告闭环，形成针对比赛（特别是服务外包属性）的降维打击。

## 开工前提

进入本阶段前，默认以下事项已经完成并验收：

- `Phase 4` 多模型骨架搭建已完成。
- 系统前后端统一数据协议已经稳定，可承载扩展字段（如 `inference_ms` 分流、过滤项标识）。
- 前端基础架构升级为 `Next.js 16` / `React 19` 等最新生态，UI 规范（黑色流体美学）已奠基。

## 范围

本阶段优先承诺落地以下能力：

- **智能大语言模型 (LLM) 诊断接入**
- **交互式 GSD 参数 (物理像素转换)** 以及准确的结构化度量（掩膜长宽面积）
- **性能耗时细粒度拆分展示**（证明模型侧耗时低于 200ms）
- **抗干扰演示项过滤图层展现**
- **高品质前端 PDF/图文报告一键合成模块**

## 实施顺序

1. 先跑通外部 LLM API 分析链路并在侧边栏实装（见效最快、产品提升感最强）。
2. 再补上 GSD 与实际物理空间运算的前后端逻辑（赛题硬骨头）。
3. 开发前端一键检验报告排版及输出（形成服务闭环）。
4. 增补可视化效果（耗时拆解、灰度干扰项显示）。

## 关键约束

- **不卷纯算法指标**：不再花主要时间纠结于微调 YOLO 提升一两点 mAP。重心全面转向全家桶式的功能工程集成。
- **降本增效**：大模型 API 可以优先使用成本低、效果好的第三方模型，只需对接标准 REST API 即可，无需本地私有化部署庞大的 LLM。
- **前端排版**：报告必须直接能给非技术人员（桥梁巡检外包客户）看懂。

## 完成定义

满足以下条件视为 `Phase 5` 完成：

- 算法跑完一张图后，能自动调用 LLM 吐出一段符合业务专家口吻的文本。
- 前台可以滑动设置焦距/转换系数，底部的病害大小能从像素精确转为毫米(mm)。
- 系统能输出排版精美的 A4 尺寸报告供用户下载打印，该报告包含前序所有产出（原图、识别结果、数据表格、LLM分析）。
