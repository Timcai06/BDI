# LESSONS

## 当前阶段经验

### 1. 先定方向，再定实现

如果项目方向没有先收敛，后面的前后端和算法对接会不断返工。

### 2. 系统不能绑定单一权重文件

项目的核心不是某一个 YOLOv8 权重，而是一个可兼容 YOLOv8 及其优化变体的推理框架。

### 3. 比赛更看重系统化表达

赛题不仅看模型指标，也看“核心算法系统”和“实际应用效果展示”。

### 4. 前后端的价值在于把算法变成作品

真正的差异化不只是把模型跑通，而是把结果展示、导出、解释和演示做完整。

### 5. 算法升级不能破坏产品接口

算法优化应被视为模型层和适配层的变化，而不是前后端协议的变化。只要输出协议稳定，系统就能持续演进而不频繁返工。

### 6. Mock 闭环能显著降低联调风险

在真实模型接入前，先把统一 schema、错误结构和前后端状态流跑通，可以把“协议问题”和“模型问题”拆开处理，联调效率会高很多。

### 7. 真实模型依赖应与基础开发依赖解耦

像 `ultralytics` 这类重型依赖，不适合强绑在所有基础环境里。拆成可选安装后，前后端骨架开发、测试和协议联调就不会被模型依赖阻塞。

### 8. Python 版本会直接影响模型接入节奏

当前机器的 `Python 3.14` 可以支撑基础 FastAPI 开发，但 `ultralytics` 依赖的 `torch` 暂时无法直接安装，因此真实模型接入建议单独使用 `Python 3.9` 到 `Python 3.12` 的虚拟环境。

### 9. “代码已落位” 不能等同于 “功能已完成”

像真实 runner 工厂、adapter 和前端统一 schema 这类能力，即使代码已经写好，也必须区分：

- 代码已存在
- mock 路径已验证
- 真实权重场景已验证

只有最后一层也通过，才能把该项记为真正完成。否则文档会比实际进度超前。

### 10. 真实模型环境需要锁定依赖组合

这次验证里，`.venv-yolo` 虽然已经安装了 `ultralytics`，但 `torch 2.2.2` 与 `NumPy 2.0.2` 组合会触发导入兼容问题。对于真实模型联调，除了记录“装了什么包”，还需要把关键版本约束固化到依赖文件里。

### 11. 沙箱内的模型报错不一定等于项目报错

这次 `torch` 在沙箱内触发了 OpenMP 共享内存错误，但在沙箱外验证后，`ultralytics + torch` 可以正常导入。涉及底层运行时的模型依赖时，需要把“沙箱限制”和“项目真实故障”区分开。

### 12. 阶段完成后必须立即关账

当真实联调已经完成时，不能继续让 `README`、`plan/` 和 `AGENTS/` 停留在“未完成”状态。阶段结论一旦落地，就要立刻同步入口、计划和记忆文档，否则团队后续会在错误前提上继续规划。

### 13. AGENTS 文档应始终先于代码被读取

这个项目的执行规范、阶段状态和架构约束都集中在 `AGENTS/`。无论是继续开发、修 bug 还是做阶段判断，都应先以 `AGENTS/` 为第一参考源，再进入代码和功能实现。

### 14. Phase 2 完成不等于展示能力完成

真实单图链路跑通后，说明 MVP 闭环已经成立，但“可展示、可解释、可回看”的体验仍然属于下一阶段任务。阶段切换的关键是承认基础链路已闭环，同时把展示增强和历史能力留给 `Phase 3` 来系统完成。

### 15. 历史回看应先复用统一结果文件

`Phase 3` 的历史结果能力不需要一开始就上数据库。先基于本地 `results/*.json` 建立列表和详情读取接口，前端就能快速复用现有结果页完成回看，这种路径最稳，也最符合当前阶段约束。

### 16. 导出能力优先按介质拆分

JSON 导出最稳的方式是直接基于前端当前 `result` 序列化下载，不依赖后端静态文件暴露；overlay 导出则更适合通过独立接口返回文件。两者拆开处理，能明显降低浏览器访问本地路径带来的不确定性。

### 17. 用户真正需要的是“记录工作台”而不是“上传表单”

一旦系统开始积累分析结果，UX 的核心就不再是上传动作本身，而是让用户持续看到、打开、比较和继续处理已有记录。历史列表、原图预览、回看和结果后的下一步动作，优先级会高于继续堆上传表单细节。

### 18. 首页必须在未上传时也传达“系统里已经有什么”

如果首页只有上传框，用户会把系统理解成一次性工具。把最近记录、推荐下一步和历史入口提前到首页，用户会更自然地理解这个系统是可以持续复查和回看的。

### 19. “新建分析”更适合作为动作而不是页面

当产品已经有主页和分析记录页时，“新建分析”应被建模为一个高优先级动作入口，而不是再占用一个长期导航页面。弹出式分析面板更符合用户心智，也能让结果页只在真正需要时出现。
