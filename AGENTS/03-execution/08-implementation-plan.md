## 阶段四：兼容优化模型与批量能力

- 先增加模型版本注册机制
- 先补齐统一 runner / adapter 与结果适配骨架
- 再增加模型版本切换
- 再增加批量任务
- 再增加切片推理模式
- 继续增加量化字段

当前结论：

- `Phase 4` 已启动
- 模型注册、runner 管理与统一切换骨架已完成第一轮落地
- 前端已支持模型版本选择与基础模型对比
- 当前最重要的工作不再是“先搭骨架”，而是用第二个真实模型版本验证这套骨架

当前对 `Phase 4` 的执行顺序建议是：

1. 先接入第二个真实模型版本
2. 再验证“换权重 + 少量配置”的接入流程
3. 再补更清晰的模型可用性提示
4. 最后推进批量任务与 `sliced` 推理

原因：

- 当前抽象骨架已经具备雏形，下一步要验证它是否真能承接新的真实模型版本
- 先完成真实验证，可以更早发现模型配置、runner 和输出适配层里的隐藏问题
- 批量和切片属于能力扩展，仍然应建立在统一推理骨架与真实模型验证基础之上
