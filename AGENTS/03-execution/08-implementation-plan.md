# IMPLEMENTATION PLAN

## 阶段一：文档与结构

- 明确项目核心目标
- 明确产品需求
- 明确前后端与算法边界
- 明确接口协议与兼容策略

## 阶段二：MVP 闭环

- 完成单张图像上传
- 接入当前 YOLOv8 推理
- 返回统一结果 JSON
- 展示原图与 Mask 叠加
- 支持结果导出

当前结论：

- `Phase 2` 已完成并通过真实单图链路验收
- 当前实施阶段为 `Phase 3`

## 阶段三：结果展示增强与系统稳定化

- 完善结果页
- 增加病害详情展示
- 增加筛选与联动交互
- 增加 overlay / JSON 导出入口
- 增加统一错误状态展示
- 增加历史结果基础能力

## 阶段四：兼容优化模型与批量能力

- 先增加模型版本注册机制
- 先补齐统一 runner / adapter 与结果适配骨架
- 再增加模型版本切换
- 再增加批量任务
- 再增加切片推理模式
- 继续增加量化字段

当前对 `Phase 4` 的执行顺序建议是：

1. 先完成模型注册与统一推理接口
2. 再稳定统一输出协议
3. 然后接入至少两个模型版本
4. 最后推进批量任务与 `sliced` 推理

原因：

- 模型抽象先完成，后续算法同学交付新权重或新版本时，系统接入成本才会真正降低
- 结果协议先稳定，前端和导出能力才不会被后续模型升级频繁打断
- 批量和切片属于能力扩展，应建立在统一推理骨架之上

## 阶段五：比赛展示优化

- 完善演示页面
- 完善结果说明
- 完善性能与对比展示
- 完善最终交付表达
